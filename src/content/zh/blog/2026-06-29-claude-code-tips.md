# Claude Code 实用技巧分享

用了一段时间 Claude Code，总结一些真正好用的技巧。

## 1. 快速启动

不用每次都打完整命令，设置个 alias：

```bash
alias cc="claude"
```

以后直接 `cc` 就能启动。

## 2. 用 Markdown 组织规则

`CLAUDE.md` 里用清晰的标题和列表，Claude 理解得更好：

```markdown
# 安全规则
- 不硬编码密钥
- 用户输入必须校验

# 代码风格
- 2 空格缩进
- 优先 const
```

> **提示**：规则文件不用太长，20-30 行精炼的规则比 100 行冗长的描述更有效。

## 3. 善用文件引用

直接告诉 Claude 文件名，它会自己去读：

```
看看 src/config.ts 里的数据库配置，帮我加上连接池参数
```

## 4. 让 Claude 先理解再动手

复杂任务先让它分析：

```
先看看这个项目的整体结构，然后告诉我怎么添加用户认证功能
```

这样 Claude 会先了解项目，再给出更合理的方案。

## 5. 批量操作

需要改多个文件时，一次性说清楚：

```
把所有 API 接口的返回格式统一成 { code, data, message } 的结构
```

Claude 能跨文件批量修改，比手动一个个改快得多。

## 6. 代码生成后要审查

> **注意**：AI 生成的代码一定要自己看一遍。重点检查：安全漏洞、边界条件、错误处理。

## 7. 建立个人 Skill 库

把常用操作写成 Skill，比如：

- 创建新页面的模板
- 数据库迁移的标准流程
- 部署检查清单

用得越多，效率越高。

## 总结

Claude Code 是个强大的工具，但关键是找到适合自己的工作流。多试、多总结，你会发现越来越多的用法。
