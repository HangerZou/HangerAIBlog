# 进阶技巧与性能优化

掌握基础用法后，本文介绍进阶技巧，让你的 AI 编程效率更上一层楼。

## 上下文管理

### 精准添加上下文

```bash
# 添加特定文件
claude add src/api/user.ts

# 添加整个目录
claude add src/components/

# 添加匹配的文件
claude add "*.test.ts"
```

### 清除上下文

```bash
# 清除当前对话
/clear

# 清除特定文件
claude remove src/old-file.ts
```

### .claudeignore 文件

创建 `.claudeignore` 排除不需要的文件：

```gitignore
# 依赖
node_modules/

# 构建产物
dist/

# 大文件
*.mp4
*.zip

# 敏感文件
.env
.env.local
```

## 工作流优化

### 任务分解

大任务拆成小步骤：

```bash
# 第一步：设计
"设计用户认证的数据库表结构"

# 第二步：实现
"根据表结构创建 API 接口"

# 第三步：测试
"为 API 接口编写单元测试"

# 第四步：集成
"在前端页面集成认证功能"
```

### 检查点机制

每完成一步都检查：

```
检查刚才的代码：
1. TypeScript 类型是否正确
2. 是否有潜在错误
3. 是否符合项目规范
```

### 并行任务

对于独立的任务，可以并行执行：

```
同时创建以下三个组件：
1. UserCard.vue - 用户卡片
2. UserList.vue - 用户列表
3. UserForm.vue - 用户表单
```

## 性能优化

### 模型选择策略

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 简单修改 | Haiku | 速度快 |
| 常规开发 | Sonnet | 平衡 |
| 复杂架构 | Opus | 能力强 |

```bash
# 切换模型
claude config set model claude-haiku-4-5-20251001
```

### 减少 Token 消耗

1. **精准描述**：避免冗余信息
2. **分步执行**：一次只做一件事
3. **清除历史**：定期 `/clear`
4. **使用 .claudeignore**：排除无关文件

### 缓存利用

```bash
# 启用缓存
claude config set cache true

# 清除缓存
claude cache clear
```

## 高级功能

### 自定义命令

创建 `.claude/commands/` 目录下的命令：

```markdown
# .claude/commands/review.md

审查以下代码：
- 检查 TypeScript 类型
- 检查安全问题
- 检查性能问题
- 给出改进建议
```

使用：
```bash
/review src/api/user.ts
```

## 安全实践

### 敏感信息处理

```
检查代码中是否有硬编码的：
- API Key
- 密码
- 数据库连接字符串
```

### 依赖安全

```
检查 package.json 的依赖：
- 是否有已知漏洞
- 是否需要更新
- 是否有不必要的依赖
```
