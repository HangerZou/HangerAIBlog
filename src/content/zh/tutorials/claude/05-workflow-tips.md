# 高效工作流

掌握了 Claude 的基本用法后，如何让协作更高效？本文分享实用的工作流技巧。

## 技巧一：描述要具体

```
❌ "帮我写个登录功能"
✅ "用 Vue 3 + Element Plus 写一个登录表单，包含手机号和密码字段，
   密码用 RSA 加密传输，登录失败显示错误提示"
```

描述越具体，Claude 生成的代码越符合你的需求。

## 技巧二：分步骤完成复杂任务

不要一次性让 Claude 做太多事。把大任务拆成小步骤：

```bash
# 第一步：先创建数据模型
"帮我设计用户表的 TypeScript 类型"

# 第二步：再写接口
"根据这个类型创建 API 接口"

# 第三步：最后写页面
"用这个接口实现用户列表页面"
```

> **提示**：每完成一步，检查一下结果再继续下一步，这样出错更容易定位。

## 技巧三：善用上下文

Claude Code 能读取你项目里的文件。直接引用文件名：

```
看看 src/utils/auth.ts 这个文件，帮我加上 token 刷新逻辑
```

不需要手动复制代码，Claude 会自己去读。

## 技巧四：让 Claude 解释代码

遇到不懂的代码，直接问：

```
解释一下 src/core/parser.ts 里的 parseExpression 函数是怎么工作的
```

比自己看代码快多了。

## 技巧五：代码审查

写完代码后让 Claude 审查：

```
审查一下 src/api/user.ts 的代码，看看有没有安全问题或可优化的地方
```

Claude 会从安全、性能、可读性等角度给出建议。

## 技巧六：自动化重复工作

对于经常做的事情，创建 Skill 或脚本：

```bash
# 把常用操作封装成 Skill
.claude/skills/create-component.md
.claude/skills/create-api.md
.claude/skills/deploy.md
```

> **注意**：AI 是工具不是替代品。理解生成的代码，保持自己的判断力，这才是高效协作的关键。

## 小结

高效工作流的核心：**具体描述、分步执行、善用上下文、持续迭代**。把这些习惯融入日常开发，你会发现效率提升显著。
